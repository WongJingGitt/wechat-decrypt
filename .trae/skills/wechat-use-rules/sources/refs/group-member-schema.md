# 群成员画像规范

本文件定义群成员画像的结构和写入规则，供 `group-builder.md` 在合并阶段引用。

> ⛔ **画像文件必须由 LLM 亲自撰写**
> 任何脚本均不得直接写入 `experience/members/{wxid}.md`。
> 脚本产出的数字是素材，LLM 读取后独立分析、归纳、写入，这个过程不可省略。

与 `contact-schema.md`（私聊好友）的核心区别：
- 私聊画像以**双边关系**为主轴，数据直接反映你与TA的关系
- 群成员画像以**群内行为**为主轴，关注TA在群内的角色、网络位置和行为模式

## 存储位置

```
experience/members/{wxid}.md
```

一人一文件，跨群。同一成员在多个群的证据汇聚在同一文件，来源用 group_wxid 标注。

## 写入规则

### 优胜略汰机制
- **特质/风格上限 5 个**：满员时新特质须比现有最弱者更核心才能替换，否则归并到最接近的已有特质
- **证据上限 3 条**：满员时新证据须比**同来源**最弱者更典型才能替换
- **禁止跨群替换**：来自 `[groupA]` 的证据不能被 `[groupB]` 的证据挤掉
- **跨群矛盾升格**：同一特质在不同群表现相反时，合并为高阶特质，标注「跨群」，保留两边证据

### ⭐ 强度评级标准
- ⭐⭐⭐：显著偏离基线，或极为典型的代表性行为
- ⭐⭐  ：中度偏离基线，或有一定代表性
- ⭐   ：轻微信号，作为补充证据

### 里程碑事件写入规则
- 必须同时满足：①单次事件（非日常互动）；②对该成员在群内的位置/关系有实质性影响
- 上限 10 条，满员时新事件须比现有最老条影响更大才能替换
- 若无符合条件的事件，此节留空
