# 联系人画像规范（私聊好友）

本文件定义私聊联系人画像的结构和写入规则，供 `profile-builder.md` 在合并阶段引用。

> ⛔ **画像文件必须由 LLM 亲自撰写**
> 任何脚本均不得直接写入 `experience/contacts/{wxid}.md`。
> 脚本产出的数字是素材，LLM 读取后独立分析、归纳、写入，这个过程不可省略。

## 存储位置

```
experience/contacts/{wxid}.md
```

## 文件结构

```markdown

- 其他称呼: xxx、xxx（从聊天记录中提取——对方自称、你称呼对方、第三方提及时用的代指，均收录）
- 性别/年龄感: 男，30s

## 基线指标

> 跨源出现矛盾时合并为高阶特质，标注「跨源」
```

## 写入规则

### 优胜略汰机制
- **特质/风格上限 5 个**：满员时新特质须比现有最弱者更核心才能替换，否则归并到最接近的已有特质
- **证据上限 3 条**：满员时新证据须比最弱者更典型才能替换
- **不标注 wxid 来源**：私聊来源唯一，所有描述以"我"第一人称视角呈现
- **跨源矛盾升格**：同一特质在不同来源表现相反时，合并为更高阶特质，标注「跨源」，保留两边证据

### ⭐ 强度评级标准
- ⭐⭐⭐：显著偏离基线，或极为典型的代表性行为
- ⭐⭐  ：中度偏离基线，或有一定代表性
- ⭐   ：轻微信号，作为补充证据

### 里程碑事件写入规则
- 必须同时满足：①单次事件（非日常互动）；②对双方关系有实质性影响
- 上限 10 条，满员时新事件须比现有最老条影响更大才能替换
- 若无符合条件的事件，此节留空，不要随意填写
