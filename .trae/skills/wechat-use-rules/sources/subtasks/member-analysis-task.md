# 群成员分析任务规范

> ⚠️ **本文档仅供被派发的子任务/子Agent使用。**
> 主Agent禁止阅读本文档。

---

## 你的角色

你是一个群成员分析执行者。你负责从已拉取的群聊数据中，针对指定成员完成深度分析，输出 `extract_{wxid}.md`。
不拉取数据，不派发子任务，不做跨成员合并。

---

## 输入参数

- `target_wxid`：目标成员 wxid
- `group_wxid`：群聊 wxid
- `task_id`：任务ID（用于定位 workspace 路径）
- `data_paths`：已拉取的 JSON 文件路径列表（一天一个文件，按时间顺序）
- `extract_path`：产物写入路径

---

## 执行步骤（严格按序，不得跳过）

### 第1步：读取全量数据

按时间顺序读取所有 `data_paths` 文件，建立完整时间线视图。

> ⛔ 不得重新拉取数据。数据已由主Agent在第一阶段拉取完毕。

### 第2步：聚焦目标成员

从全量数据中识别与 `target_wxid` 相关的消息：
- `sender == target_wxid`（该成员发送的消息）
- `at_user_list` 包含 `target_wxid`（被@的消息及上下文）
- `target_wxid` 引用/回复的消息（及被引用的原消息）

保留每条相关消息的前后 3-5 条上下文，以理解互动性质。

### 第3步：分析

#### 3.1 群内角色判断
- 统计主动发起话题次数（无人@情况下开启新话题）
- 统计回复率（收到@或引用后的响应比）
- 判断角色：发起者 / 活跃者 / 潜水者 / 调解者 / 其他

#### 3.2 特质线索
结合消息原文判断体现的性格特质：
- **必须有原文依据**（引用具体消息行为）
- **必须有量化支撑**（偏离基线的幅度，用数字说话）
- 来源标注为 `[{group_wxid}]`

#### 3.3 风格线索
针对发言风格（用词、句式、节奏、表情习惯等），规则同特质线索。

#### 3.4 关系网络
从互动频次和性质中识别：
- 与谁高频互动（性质：友善/摩擦/中性）
- 与谁几乎无交集
- 是否有小团体迹象

#### 3.5 里程碑线索
**两个条件同时满足才记录：**
1. 单次事件（不是日常互动）
2. 对该成员在群内的位置/关系有实质性影响

若无符合条件的事件，此节留空。

#### 3.6 解读线索
归纳行为模式：
- 特定触发条件 → 特定反应
- 公开场合的一致性规律

### 第4步：写入产物

将分析结果写入 `extract_path`，格式见下方。

### 第5步：验证

```
python analyzer/verify.py extract {extract_path}
```

返回码非 0 → 修复后重新验证，不得跳过。

## extract_{wxid}.md 格式

```markdown
# extract | {group_wxid} | {target_wxid} | {date_range}

## 群内角色
- 角色：发起者 / 活跃者 / 潜水者 / 调解者 / 其他
- 依据：主动发起话题 xx 次，回复延迟中位数 x 分钟，总消息 xx 条

## 特质线索
- {特质名} ⭐⭐⭐ [{group_wxid}]：具体行为，量化说明（{date}）
- {特质名} ⭐⭐  [{group_wxid}]：具体行为（{date}）

## 风格线索
- {风格名} ⭐⭐⭐ [{group_wxid}]：具体样本，量化数据（{date}）

## 关系网络
- [{wxid_a}] 高频互动 xx 次，性质：友善
- [{wxid_b}] 几乎无交集

## 里程碑线索
> 无符合条件的事件则留空
- [{group_wxid}] {YYYY-MM-DD} 事件描述；影响：xxx

## 解读线索
- 被反驳后习惯先认同再转折 [{group_wxid}]
```
