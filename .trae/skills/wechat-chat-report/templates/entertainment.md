# 娱乐向群聊报告生成模板

## 使用说明

**⚠️ 强约束：在生成报告前必须完整执行以下步骤**

### 数据文件路径（任务ID：自动从目录名获取）
```
data/任务ID/entertainment_data.json  # 专项分析数据（必须读取）
data/任务ID/metadata.json            # 完整元数据（必须读取）
data/任务ID/messages.json            # 原始消息 JSON（按需引用）
report/任务ID/entertainment.md       # 最终报告输出
```

### 强制阅读步骤
1. **先读取** `entertainment_data.json` 的 `tips.must_read`，按要求读取经验库和原始记录
2. **再读取** `entertainment_data.json` 的 `basic`、`rankings`、`events`、`sender_stats`
3. **按需读取** `metadata.json` 补充细节（top_words、interaction_matrix 等）
4. ⛔ **必须读取 `messages.json` 原始消息全文**，用于：名场面还原原话引用、热搜榜发言截取、时间线关键事件定位。不得以聚合数据替代原文阅读。

### 引用格式（全局约定，所有原话引用必须遵守）

```
- **昵称**（M月D日 HH:MM）：「原话原文」
```

示例：
```
- **猫绪**（4月17日 15:23）：「你tm卡了24小时？？？」
```

> ⛔ 缺少时间戳、或时间戳与 messages.json 对不上的引用，会被验证脚本判为无效。发言人写昵称（不写 wxid）。

### WechaUseRules 经验库
```
../wechauserules/experience/contacts/
```
⛔ **步骤5a（生成报告前必须先完成）**：读取该目录下所有联系人画像文件，了解：
- 群成员的性格特质和发言风格
- 关系备注（和谁关系好、和谁不对付）

读取完毕后，输出 `data/任务ID/sender_map.json`：
```json
{
  "abc123wxid": "猫绪",
  "def456wxid": "🦦"
}
```
没有经验库画像的成员，用 rankings/sender_stats 中出现的标识符作为昵称。

## 提示
- 所有总结内容必须基于聊天记录，不得编造任何不存在的内容
- 总结时请务必忽略：`[动画表情]` / `[视频]` / `[图片]` / `[语音]`
- `wxid` 仅用作辅助识别，最终报告不出现任何 wxid
- **行号索引仅供内部检索用，最终报告不出现任何行号引用**
- entertainment_data.json 中的 tips.report_style 提供了报告风格指导，必须遵循

## 输出要求

### 0. 开篇数据卡片
使用 entertainment_data.json 的 basic 数据，生成统计概览：
- 总消息数、人数、时间范围
- **话题分布**：读 `messages.json` 归纳这几天聊了哪些主题（每个主题一句毒舌总结，不超过5个）
- 高峰时段（哪个小时消息最密集）

> ⛔ 禁止在数据卡片中写「内容类型分布：图片/表情/语音各多少」——LLM 无法看到实际图片和表情内容，统计这些数字毫无意义。

格式：
```
**统计周期**：X年X月X日 ~ X年X月X日（X天）
**消息总数**：XXX 条
**参与人数**：XX 人
**话题分布**：
- 【话题名】一句毒舌总结这个话题在聊什么
- 【话题名】...
```

### 1. 群聊奥斯卡颁奖典礼（不少于3个）

**奖项设置**（从 rankings 选取）：
- 最佳段子手（优先选择含撤回+回复链的发言）
- 互联网嘴替（谁的发言被复读/引用次数最多）
- 凌晨发癫榜（凌晨0-6点活跃发言统计）
- 人间唢呐奖
  唢呐指数 = 发言数 + 被回复数 + 撤回数×5
- 链接狂魔（小程序/视频/链接分享次数最多）
- 表情包依赖症（动画表情使用频率最高）
- 撤回黑洞（统计谁的撤回消息最多）
- 新人引入奖（邀请入群最多的人）
- 拍一拍之王（被拍次数最多）

**数据支撑**：
- 以"名场面还原"为标题
- ⛔ **必须读 `messages.json` 原文**，找到该发言者的代表性对话片段（含上下文），用 `ul+li` 格式逐条引用原话，再附毒舌点评
- 引用格式：
  ```
  **名场面还原**：
  - [昵称A]：「原话原文」
  - [昵称B]：「原话原文」（回复上条）
  - [昵称A]：「原话原文」
  > 毒舌点评：...
  ```
- 每个获奖者至少引用 2 段真实对话上下文，不得用叙述替代引用

**颁奖词**：毒舌化，模仿综艺吐槽

### 2. 群聊话题榜（不少于3个）

- 模仿微博热搜榜单
- 每个话题必须包含：
  - 关键参与者（谁在聊）
  - ⛔ **至少1条真实原话引用**（谁说的，从 `messages.json` 摘取），不得只写「话题热度：XX次」
  - 毒舌点评
- 结合 events.replies、events.patpats 等挖掘话题线索

### 3. 整活总结模块

- **成就系统**：发放虚拟勋章（从 rankings 选取）
  - 🌙 凌晨蹦迪王
  - 🗑️ 撤回界灭霸
  - 📊 超级唢呐
  - 🔗 链接搬运工
  - 😈 人形复读机
  - 📸 拍一拍之王
  - 🏃 跑路冠军
  > ⛔ 如果某个勋章的获得者数据为空（即「无」），直接跳过，不写入表格

- **时间线沙漏**：每天必须读 `messages.json` 找出当天的关键事件节点
  - ⛔ 每天至少 4 条时间节点，必须有具体时间（精确到小时）+ 具体发生了什么
  - 不得写「全天持续活跃」「话题全覆盖」此类空话
  - 每条节点格式：`- HH:00 [发生了什么，谁参与，高光言论或行为]`

- **明日预言局**（如有）
  - 【数据支撑】扫描高光事件/关键词
  - 【推理维度】翻车事件后续 / 未兑现承诺 / 成员行为规律
  - 【概率规则】💥 = 高(80%+) / 🌀 = 中(50%) / 🌪️ = 低(20%)

## 步骤6：验证报告（强制，非0禁止结束）

报告生成后必须运行：

```
python analyzer/verify_report.py report/任务ID/entertainment.md data/任务ID/messages.json data/任务ID/sender_map.json
```

脚本会检查：
- 每个奖项名场面引用 ≥ 2 条（含时间戳）
- 所有时间戳可在 messages.json 中核实（±1分钟窗口）
- 热搜榜每条含 ≥ 1 条原话引用
- 时间线每天 ≥ 4 条节点
- 成就表格无空获得者行
- 数据卡片含话题分布

返回码非0时，按错误提示词修改报告后重新验证。
